# TG WS Proxy

**TG WS Proxy** — локальный MTProto-прокси для Telegram Desktop, который использует WebSocket-транспорт для подключения к дата-центрам Telegram. Решение работает локально, не требует сторонних прокси-сервисов и сохраняет стандартную схему шифрования Telegram.

## Быстрый запуск в Windows

Выберите подходящую сборку под свою систему и скачайте её из страницы релизов вашего проекта.

Поддерживаемые варианты:

- `TgWsProxy_windows.exe` — Windows 10+
- `TgWsProxy_windows_arm64.exe` — Windows ARM64
- `TgWsProxy_windows_7_64bit.exe` — Windows 7 x64
- `TgWsProxy_windows_7_32bit.exe` — Windows 7 x86

После первого запуска приложение автоматически создаст локальный прокси и покажет инструкцию по его подключению. Далее программа продолжает работать через системный трей.

## Возможности меню

- **Открыть в Telegram** — открыть ссылку подключения.
- **Скопировать ссылку** — получить `tg://proxy`.
- **Перезапустить прокси** — перезапуск без закрытия приложения.
- **Настройки** — изменение параметров работы.
- **Открыть журнал** — просмотр логов.
- **Выход** — завершение работы.

## Подключение Telegram Desktop

### Автоматический вариант

Выберите пункт **«Открыть в Telegram»** в меню трея.

Если ссылка автоматически не открылась:

1. Скопируйте ссылку.
2. Отправьте её себе в «Избранное».
3. Откройте ссылку и подтвердите подключение.

### Ручная настройка

Настройки → Продвинутые настройки → Тип подключения → Прокси

Параметры:

- Тип: MTProto
- Сервер: `127.0.0.1`
- Порт: `1443`
- Secret: значение, созданное приложением.

## Как работает TG WS Proxy

```
Telegram Desktop → MTProto Proxy → WebSocket → Telegram DC
```

Во время запуска приложение поднимает локальный MTProto-прокси, определяет целевой дата-центр Telegram и устанавливает WebSocket-соединение. Если использование WebSocket недоступно, применяется альтернативный маршрут подключения.

> [!IMPORTANT]
>
> Если медиафайлы не загружаются, попробуйте оставить только запись `4:149.154.167.220` в разделе **DC → IP** либо полностью очистить список.
